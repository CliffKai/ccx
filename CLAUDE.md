# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

`ccx` is a pure POSIX shell tool for per-terminal Claude Code profile switching. It wraps the `claude` command with `--settings <profile.json>` so different terminal windows can use different API providers/configurations simultaneously, without modifying `cc-switch` or global Claude settings.

## Architecture

Two shell scripts, no build system, no tests:

- **`ccx.sh`** â€” Sourced into the user's shell (zsh/bash). Defines shell functions (`ccx_use`, `ccx_list`, `ccx_run`, etc.) and a `claude()` wrapper that auto-injects `--settings` when a profile is active. Also auto-registers shortcut functions for each profile file (e.g., `profiles/gaccode.json` â†’ `gaccode` shell function).

- **`sync_ccswitch_profiles.sh`** â€” Standalone script invoked by `ccx_sync`. Reads the `cc-switch` SQLite database (`cc-switch.db`) and `settings.json`, merges common config with per-provider overlays using `jq`, and writes profile JSONs into `profiles/`. Produces `.ccx-manifest.tsv` (aliasâ†’labelâ†’source mapping) and `.ccx-generated.list` (for cleanup on next sync).

- **`profiles/*.json`** â€” Claude settings snippets (env vars, model, effortLevel, permissions). Generated by sync or hand-edited. The `.ccx-manifest.tsv` and `.ccx-generated.list` are internal metadata files managed by the sync script.

## Key Design Details

- Profile resolution (`ccx_resolve_profile`) checks the manifest TSV first (supports non-ASCII names like Chinese via slug aliases), then falls back to matching `profiles/<name>.json` directly.
- The `claude()` shell function only injects `--settings` if the user hasn't already passed `--settings` themselves (`ccx_has_settings_arg`).
- `ccx_slugify` has hardcoded Chineseâ†’slug mappings for specific provider names; other names are lowercased and sanitized.
- External dependencies: `jq`, `sqlite3`, `base64`.

## Development

No build or test commands. To try changes:

```sh
# Load ccx into current shell
export CCX_ROOT="/Users/cliffkai/Code/claude-api"
source "$CCX_ROOT/ccx.sh"

# Sync profiles from cc-switch (requires cc-switch database)
ccx_sync

# Verify profile listing
ccx_list
```

Shell compatibility target is POSIX sh (the scripts use `#!/usr/bin/env sh`), but `ccx.sh` uses `local` (a common non-POSIX extension) and is primarily used under zsh/bash.
